## 概要

このスキルは技術的に正確で読みやすいドキュメントの作成・更新を担当する。既存ドキュメントとの整合性を保ちながら、ユーザー視点で分かりやすい説明を提供する。

## 主要機能

- 技術文書の作成（チュートリアル/リファレンス/概念説明）
- 既存ドキュメントの更新
- ドキュメントスタイル・フォーマットの一貫性維持
- 段階的な説明（初心者から上級者まで）

## 使用方法

### 5ステップワークフロー

#### Step 1: 目的とスコープ確認
- ドキュメントの目的を明確化（チュートリアル/リファレンス/概念説明）
- ターゲット読者を特定（初心者/中級者/上級者）
- スコープを決定（何を含め、何を含めないか）

#### Step 2: 既存コンテンツ分析
- `Read`で関連する既存ドキュメントを確認
- `Grep`でドキュメントスタイル・フォーマットを把握
- 重複や矛盾がないか確認

#### Step 3: 情報収集
- `Read`でコードベースから必要な情報を抽出
- `Glob`で設定ファイル、サンプルコードを確認
- `Bash`で実行結果を確認（必要に応じて）

#### Step 4: 構造設計
- 見出し構造を設計
- 論理的な流れを確認
- コード例の配置を決定

#### Step 5: 執筆・レビュー
- ドラフトを作成（`Write`または`Edit`）
- **自己レビュー**:
  - 技術的正確性を確認
  - 読みやすさを確認
  - コード例が実行可能か確認
- **問題がある場合**: Step 3（情報収集）に戻る
- **レビュー通過後**: ドキュメントを保存

## Examples

**例1: 新規チュートリアル作成**

入力:
```
"新機能のチュートリアルを作成して"
```

実行:
```
Step 1: 目的確認（初心者向けチュートリアル、スコープ: 基本的な使い方のみ）
Step 2: Read で既存チュートリアルの形式を確認
Step 3: Read と Glob で新機能のコードを確認
Step 4: 見出し構造設計（概要→前提条件→手順→例→トラブルシューティング）
Step 5: Write で docs/tutorial-new-feature.md を作成
```

出力:
```
docs/tutorial-new-feature.md に初心者向けチュートリアルを作成しました。
構成: 概要、前提条件、3つの基本手順、2つのコード例を含みます。
```

**例2: README更新**

入力:
```
"READMEにインストール手順を追加して"
```

実行:
```
Step 1: 目的確認（インストール手順の追加）
Step 2: Read で README.md の構造を確認
Step 3: Bash でインストールコマンドの動作確認
Step 4: 「## Installation」セクションを追加する位置を決定
Step 5: Edit で README.md を更新
```

出力:
```
README.md の「## Quick Start」の前に「## Installation」セクションを追加しました。
npm, yarn, pnpm の3つの方法を記載しています。
```

**例3: APIリファレンス作成**

入力:
```
"新しいAPIのリファレンスを作成して"
```

実行:
```
Step 1: 目的確認（APIリファレンス、中級者向け）
Step 2: Read で既存APIドキュメントの形式を確認
Step 3: Read でAPIコードから関数シグネチャを抽出
Step 4: 構造設計（関数名→説明→パラメータ→戻り値→例外→例）
Step 5: Write で docs/api-reference.md を作成
```

出力:
```
docs/api-reference.md に5つのAPI関数のリファレンスを作成しました。
各関数にパラメータ表、戻り値、使用例を含みます。
```

## Guidelines

### ドキュメント構造

ドキュメント種別別の構造テンプレートは [templates/structure.md](templates/structure.md) を参照。

### 重要な原則

1. **簡潔で明確な表現** - 1文1メッセージ、専門用語は初出時に説明
2. **実例とコード例を含める** - コードブロックに言語指定、実行可能なコード
3. **段階的な説明** - 基本から応用へ、前提知識を明示
4. **最新情報に基づく正確な内容** - コードとの整合性確認

### 出力形式

**Markdown形式**、**日本語**、見出し・コードブロック・リストを適切に使用

## Limitations

- ドキュメントファイル（.md）の作成・編集のみ
- ソースコードの直接編集は行わない
- ドキュメントの翻訳は対応外（日本語のみ）
