# コード品質ガイドライン

## 評価観点

### 命名規則
- 変数、関数、クラス名の一貫性
- 意図が明確に伝わる命名
- プロジェクトの命名規約への準拠

### 単一責任原則
- 関数/メソッドが単一の責任を持つか
- 1つの関数が複数の処理を担当していないか
- 適切な粒度での責任分割

### DRY原則
- コードの重複が排除されているか
- 共通処理が適切に抽出されているか
- 重複排除によるメンテナンス性向上

### エラーハンドリング
- 適切な例外処理
- エラーメッセージの明確性
- 例外の適切な伝播とキャッチ
- リソースの適切なクリーンアップ

### コメント
- 複雑なロジックへの適切なコメント
- コードの意図を説明するコメント
- 過度なコメントの回避（自己文書化コード）
- TODOやFIXMEの適切な使用
