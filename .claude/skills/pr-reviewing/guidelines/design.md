# 設計ガイドライン

## 評価観点

### 既存パターン
- プロジェクトの既存パターンとの一貫性
- アーキテクチャスタイルの統一
- コーディング規約への準拠
- フォルダ構成の一貫性

### 拡張性
- 将来の変更への対応しやすさ
- Open/Closed原則の適用
- プラグイン可能な設計
- 設定の外部化

### 依存関係
- 適切な依存関係管理
- 循環依存の回避
- 依存性注入の活用
- インターフェースへの依存

### インターフェース
- API設計の適切さ
- RESTful設計原則の遵守
- エンドポイントの命名規則
- レスポンス形式の一貫性
- バージョニング戦略
