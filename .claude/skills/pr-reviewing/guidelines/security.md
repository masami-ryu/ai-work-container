# セキュリティガイドライン

## 評価観点

### 入力検証
- ユーザー入力の適切な検証
- バリデーションルールの妥当性
- サニタイゼーション処理
- 型チェックと境界値チェック

### 機密情報
- ハードコードされた認証情報がないか
- APIキーやパスワードの適切な管理
- 環境変数の活用
- シークレット管理サービスの利用

### 認証・認可
- 適切な認証チェックが実装されているか
- 認可（アクセス制御）の適切な実装
- セッション管理のセキュリティ
- トークンの適切な検証

### 脆弱性対策
- SQLインジェクション対策
- XSS（クロスサイトスクリプティング）対策
- CSRF（クロスサイトリクエストフォージェリ）対策
- その他OWASPトップ10への対策
- 依存ライブラリの脆弱性チェック
