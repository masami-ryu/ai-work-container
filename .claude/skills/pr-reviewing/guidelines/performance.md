# パフォーマンスガイドライン

## 評価観点

### アルゴリズム効率
- 適切な時間計算量（O(n), O(log n)等）
- 不必要なネストループの回避
- 効率的なデータ構造の選択
- アルゴリズムの最適化

### 不要な処理
- 無駄なループや計算の排除
- 不要な再計算の回避
- キャッシュ可能な値の再利用
- 遅延評価の活用

### メモリ使用
- メモリリークの有無
- 過剰なメモリ使用がないか
- 大量データの適切な処理（ストリーミング等）
- リソースの適切な解放

### N+1問題
- データベースクエリの最適化
- N+1問題の回避
- バッチ処理の活用
- 適切なJOIN処理
- インデックスの活用
