# CLAUDE.md Suggesting

## 概要

このスキルはセッションで得られた新しい知見に基づいて、CLAUDE.mdを更新すべき点を提案する。プロジェクトの進化に合わせてドキュメントを最新に保つためのガイド。

## 主要機能

- 新しいワークフローの発見と提案
- 重要なコマンドや設定の抽出
- プロジェクト構造の変更の検出
- ベストプラクティスの識別
- 注意点や落とし穴の発見

## 使用方法

### 基本的な提案生成

1. **現在のCLAUDE.mdを読み込む**
   ```
   Read: /workspaces/ai-work-container/CLAUDE.md
   ```

2. **セッションコンテキストを分析**
   - 新しく発見した効率的な方法
   - 頻繁に使用したコマンド
   - 新しいディレクトリやファイル
   - 今後も使うべき良い習慣
   - 発見した落とし穴や注意事項

3. **更新提案を生成**
   - 追加すべき項目
   - 修正すべき項目
   - 削除すべき項目（古い情報）
   - 提案理由

## Examples

**例1: セッション終了時の自動提案**
```
User: "セッションが終わったので、CLAUDE.mdの更新提案を出して"
Assistant:
1. CLAUDE.mdを読み込み
2. セッション内容を分析
3. 更新提案を生成
   - 追加: 新しいスキル構造
   - 修正: プラン保存先の記述
   - 削除: なし
```

**例2: 新機能追加後の提案**
```
User: "新しいPRレビュー機能を追加したので、CLAUDE.mdに反映すべきか確認して"
Assistant:
1. CLAUDE.mdを読み込み
2. 新機能の内容を分析
3. 更新提案を生成
   - 追加: PRレビューの使用方法
   - 修正: ワークフローセクション
```

## Guidelines

### 分析観点

1. **新しいワークフロー**: 今回のセッションで発見した効率的な方法
2. **重要なコマンド**: 頻繁に使用したコマンドや設定
3. **プロジェクト構造の変更**: 新しいディレクトリやファイル
4. **ベストプラクティス**: 今後も使うべき良い習慣
5. **注意点**: 発見した落とし穴や注意事項

### 出力形式

```markdown
## CLAUDE.md 更新提案

### 追加すべき項目
- （項目があれば記載、なければ「なし」）

### 修正すべき項目
- （項目があれば記載、なければ「なし」）

### 削除すべき項目
- （古い情報があれば記載、なければ「なし」）

### 提案理由
（簡潔に説明）
```

### 提案基準

- **重要な変更のみ提案**: 些細な変更は提案しない
- **既存内容の確認**: 既にCLAUDE.mdに記載済みの内容は提案しない
- **新規情報のみ**: 新しい情報がない場合は「更新不要」と回答
- **ユーザー確認**: 提案後、ユーザーに確認を求める

## Limitations

- CLAUDE.mdへの直接編集は行わない（提案のみ）
- セッションコンテキストに依存（長時間のセッションほど提案が多い）
- ユーザーの明示的な承認が必要
