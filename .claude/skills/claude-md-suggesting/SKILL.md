## 概要

このスキルはセッションで得られた新しい知見に基づいて、CLAUDE.mdを更新すべき点を提案する。プロジェクトの進化に合わせてドキュメントを最新に保つためのガイド。

## 主要機能

- 新しいワークフローの発見と提案
- 重要なコマンドや設定の抽出
- プロジェクト構造の変更の検出
- ベストプラクティスの識別
- 注意点や落とし穴の発見

## 使用方法

### 基本的な提案生成

1. **現在のCLAUDE.mdを読み込む**
   ```
   Read: /workspaces/ai-work-container/CLAUDE.md
   ```

2. **セッションコンテキストを分析**
   - 新しく発見した効率的な方法
   - 頻繁に使用したコマンド
   - 新しいディレクトリやファイル
   - 今後も使うべき良い習慣
   - 発見した落とし穴や注意事項

3. **更新提案を生成**
   - 追加すべき項目
   - 修正すべき項目
   - 削除すべき項目（古い情報）
   - 提案理由

## Examples

**例1: セッション終了時の提案**

入力:
```
"セッションが終わったので、CLAUDE.mdの更新提案を出して"
```

実行:
```
1. Read で CLAUDE.md を読み込み
2. セッション内容を分析
   - 新しく追加したスキル構造を確認
   - 頻繁に使用したコマンドパターンを確認
3. 更新提案を生成
```

出力:
```markdown
## CLAUDE.md 更新提案

### 追加すべき項目
- スキル構成: .claude/skills 配下の6つのスキル説明
- 評価システム: evaluations/ ディレクトリの目的と使用方法

### 修正すべき項目
- プラン保存先: ai/plans/ のファイル命名規則を明確化

### 削除すべき項目
- なし

### 提案理由
新しいスキルベースのワークフローが確立されたため、CLAUDE.mdに記載して
今後のセッションで一貫した使用方法を維持する。
```

**例2: 新機能追加後の提案**

入力:
```
"新しいPRレビュー機能を追加したので、CLAUDE.mdに反映すべきか確認して"
```

実行:
```
1. Read で CLAUDE.md を読み込み
2. Grep で新機能の実装を確認
3. 既存のCLAUDE.md内容と照らし合わせ
```

出力:
```markdown
## CLAUDE.md 更新提案

### 追加すべき項目
- PRレビュー機能: pr-reviewing スキルの使用方法
- レビュー結果保存先: ai/reviews/ ディレクトリの説明

### 修正すべき項目
- ワークフローセクション: PRレビューフローを追加

### 削除すべき項目
- なし

### 提案理由
新しいPRレビュー機能が追加されたため、使用方法とレビュー結果の
保存場所をCLAUDE.mdに明記する必要がある。
```

## Guidelines

### 分析観点

1. **新しいワークフロー**: 今回のセッションで発見した効率的な方法
2. **重要なコマンド**: 頻繁に使用したコマンドや設定
3. **プロジェクト構造の変更**: 新しいディレクトリやファイル
4. **ベストプラクティス**: 今後も使うべき良い習慣
5. **注意点**: 発見した落とし穴や注意事項

### 出力形式

```markdown
## CLAUDE.md 更新提案

### 追加すべき項目
- （項目があれば記載、なければ「なし」）

### 修正すべき項目
- （項目があれば記載、なければ「なし」）

### 削除すべき項目
- （古い情報があれば記載、なければ「なし」）

### 提案理由
（簡潔に説明）
```

### 提案基準

- **重要な変更のみ提案**: 些細な変更は提案しない
- **既存内容の確認**: 既にCLAUDE.mdに記載済みの内容は提案しない
- **新規情報のみ**: 新しい情報がない場合は「更新不要」と回答
- **ユーザー確認**: 提案後、ユーザーに確認を求める

## Limitations

- CLAUDE.mdへの直接編集は行わない（提案のみ）
- セッションコンテキストに依存（長時間のセッションほど提案が多い）
- ユーザーの明示的な承認が必要
