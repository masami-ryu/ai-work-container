# CLAUDE.md Suggesting

## 参照ガイド

このスキルには以下の詳細ガイドラインが用意されています。必要に応じて参照してください:

- [提案基準の詳細](guidelines/suggestion-criteria.md): 提案すべきケース、分析観点、品質チェックリスト
- [良い例・悪い例](examples/good-bad-examples.md): 具体的な提案例と判定基準

---

## 概要

このスキルはセッションで得られた新しい知見に基づいて、CLAUDE.mdを更新すべき点を提案します。プロジェクトの進化に合わせてドキュメントを最新に保つためのガイドです。

**重要**: このスキルはCLAUDE.mdへの直接編集は行わず、提案のみを行います。

---

## 5ステップワークフロー

このチェックリストをコピーして進捗を追跡してください:

```
CLAUDE.md更新提案 進捗:
- [ ] Step 1: CLAUDE.mdを読み込む
- [ ] Step 2: セッションコンテキストを分析する
- [ ] Step 3: 既存内容と照らし合わせる
- [ ] Step 4: 更新提案を生成する
- [ ] Step 5: 品質チェックを実施する
```

### Step 1: CLAUDE.mdを読み込む

```
Read: /workspaces/ai-work-container/CLAUDE.md
```

**チェックポイント**:
- [x] CLAUDE.md全文を読み込み完了
- [x] 既存のセクション構成を把握
- [x] 重複確認の準備完了

---

### Step 2: セッションコンテキストを分析する

以下の5つの観点で分析:

1. **新しいワークフロー**: 繰り返し使用された作業パターン
2. **重要なコマンド**: 3回以上使用された複雑なコマンド
3. **プロジェクト構造の変更**: 新しいディレクトリやファイル
4. **ベストプラクティス**: 品質向上やエラー削減に寄与した方法
5. **注意点**: 発見した落とし穴や問題

詳細な分析方法は [guidelines/suggestion-criteria.md#分析観点の詳細](guidelines/suggestion-criteria.md#分析観点の詳細) を参照してください。

**チェックポイント**:
- [x] 5つの観点すべてで分析を実施
- [x] 新規性のある項目を抽出
- [x] 具体的な内容を記録（スキル名、コマンド、ディレクトリパス）

---

### Step 3: 既存内容と照らし合わせる

```
照らし合わせチェックリスト:
- [ ] Step 2で抽出した項目がCLAUDE.mdに記載されていないか確認
- [ ] 類似の記載がある場合は重複として除外
- [ ] 新規情報のみをリストアップ
```

**チェックポイント**:
- [x] 重複項目を除外完了
- [x] 新規情報のみをリストアップ
- [x] 新規情報がない場合は「更新不要」と判断

---

### Step 4: 更新提案を生成する

**出力形式**:
```markdown
## CLAUDE.md 更新提案

### 追加すべき項目
- （具体的な項目、なければ「なし」）

### 修正すべき項目
- （具体的な項目、なければ「なし」）

### 削除すべき項目
- （古い情報、なければ「なし」）

### 提案理由
（簡潔に説明、1〜2文）
```

**重要な原則**:
- **具体性**: 「〜について」ではなく、スキル名、コマンド、ディレクトリパスを明記
- **簡潔性**: 重要な変更のみに絞る
- **明確性**: ユーザーが理解しやすい形式

詳細は [guidelines/suggestion-criteria.md#提案すべきケース](guidelines/suggestion-criteria.md#提案すべきケース) を参照してください。

**チェックポイント**:
- [x] 各項目が具体的（スキル名、コマンド、ディレクトリパスを含む）
- [x] 重要な変更のみに絞られている
- [x] 提案理由が明確で説得力がある

---

### Step 5: 品質チェックを実施する

```
品質チェックリスト:
- [ ] 追加すべき項目が具体的で明確
- [ ] 修正すべき項目が既存セクションと対応
- [ ] 削除すべき項目が古い情報として妥当
- [ ] 提案理由が簡潔で説得力がある
- [ ] ユーザーが理解しやすい形式
- [ ] 重要な変更のみに絞られている
```

**検証パス後**: 提案をユーザーに提示

**検証失敗時**: Step 4に戻り、提案を修正

## クイックリファレンス

### 提案基準（要約）

**提案すべき**:
- 3回以上使用された複雑なコマンド
- 新しく追加された永続的なディレクトリ
- エラー削減に寄与したベストプラクティス
- 再発可能性のある問題と回避方法

**提案すべきでない**:
- 1回だけ使用したコマンド
- 既にCLAUDE.mdに記載済みの内容
- 一時的な実験やデバッグのための作業

詳細は [guidelines/suggestion-criteria.md](guidelines/suggestion-criteria.md) を参照してください。

---

## 制限事項

- **提案のみ**: CLAUDE.mdへの直接編集は行わない
- **セッション依存**: セッションコンテキストに依存（長時間のセッションほど提案が多い）
- **ユーザー承認必要**: 提案後、ユーザーの明示的な承認が必要
