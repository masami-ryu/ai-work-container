# Express ワークフロー（簡易プラン）

## 適用条件

- 変更対象が2ファイル以下
- 影響範囲が限定的
- 技術的な複雑さが低い

## 適用例

- READMEの更新
- 1-2ファイルの変更
- ドキュメントの追加・修正
- 軽微なバグ修正

## プロセス

### 1. 情報収集

- `Grep`/`Glob`で対象ファイルを特定
- 既存の関連ファイルを確認
- 必要最小限の情報のみ収集

### 2. プラン作成

- 簡潔なプランを作成
- 必須セクションのみ含める
- 完了条件を明記

### 3. 出力

- `ai/plans/YYMMDD_[概要].md`に保存
- ステータス: Draft

## 品質チェックリスト

Expressワークフローでは、プラン作成直後に1回のみ検証:

- [ ] すべての要件がプランに反映されているか
- [ ] 各タスクは具体的なアクション動詞で始まるか
- [ ] 完了条件は客観的に判断可能か
- [ ] タスク間の依存が正しく反映されているか

## 出力例

```markdown
# README更新プラン

作成日: 2025年12月20日
作成者: Plan Creator
ステータス: Draft

## 1. 概要
### 目的
READMEに新機能のドキュメントを追加する。

### スコープ
- 対象: README.md
- 対象外: 他のドキュメントファイル

## 2. 要件と制約
| ID | 種別 | 内容 | 優先度 |
|----|------|------|--------|
| REQ-001 | 要件 | 新機能の使用方法を追加 | 高 |

## 3. 実装ステップ
| タスクID | 内容 | 完了条件 | 状態 |
|----------|------|---------|------|
| TASK-001 | 使用方法セクションを追加 | セクションが存在する | [ ] |

## 4. 成功基準
- [ ] 新機能の使用方法が記載されている
```
